## 特点
- 栈是先进后出的

## 栈组成
-[ ] 线程执行时必须分配一块专用内存区域（栈）-每个线程有自己独立的栈
-[ ] 栈由多个栈帧组成（Frame Stack）
-[ ] 方法调用中正在执行的方法就是活动栈帧
-[ ] 栈的大小可以通过 -Xss 参数设置

## 栈帧
- 每个方法运行时需要的内存
- 完整的栈帧结构包括：
  - 局部变量表(Local Variables) - 存储方法参数和局部变量
  - 操作数栈(Operand Stack)- 存储计算过程中的临时操作数
  - 动态链接(Dynamic Linking) - 指向运行时常量池的方法引用
  - 方法返回地址 (Return Address) - 方法正常或异常退出后的返回位置

### 详细解释
1. 线程的独立性
   - 每个Java线程都有自己独立的栈空间
   - 不同线程的栈空间相互隔离，互不干扰
   - 这保证了线程安全，避免了多线程间的内存冲突

2. 运行时功能
- 栈内存用于存储线程执行时的关键信息：
  - 方法调用链：记录方法的调用顺序（先进后出）
  - 局部变量：存储方法中的局部变量和参数
  - 操作状态：保存方法执行过程中的中间计算结果
  - 返回地址：记录方法执行完成后应该返回的位置
  
3. 生命周期
   - 栈内存随着线程的创建而分配
   - 随着线程的结束而自动释放
   - 不需要垃圾回收机制参与管理

4. 实际应用
  - 例如，当一个线程执行 methodA() 调用 methodB() 时：
    1. 线程栈中会创建 methodA 的栈帧
    2. 在 methodA 中调用 methodB 时，创建 methodB 的栈帧
    3. methodB 执行完成后，其栈帧被弹出
    4. 继续执行 methodA 剩余的代码


### 问题辨析
**1. 垃圾回收是否涉及栈内存？**
> 结论： GC不涉及栈内存

**原因**

- 栈内存随方法调用自动分配，方法结束自动释放

**2. 栈是设置越大越好吗？**
> 结论：不是越大越好，栈设置的越大，由于机器的内存是固定的，堆的空间就变小了，更容易发生OOM


**3. 方法的局部变量是线程安全的吗？**
- 如果方法内部的变量没有逃离方法的作用范围，它是线程安全的
- 如果这个局部变量引用了对象，或逃离方法的作用，需要考虑线程安全问题

### 案例代码块
```java
public static void main(String[] args) {
    StringBuilder sb = new StringBuilder();
    sb.append(4);
    sb.append(5);
    sb.append(6);
    new Thread(()->{
        m2(sb);
    }).start();
}

// 方法的局部变量，同时没有返回，是线程安全的
public static void m1(){
    StringBuilder sb = new StringBuilder();
    sb.append(1);
    sb.append(2);
    sb.append(3);
    System.out.println(sb.toString());
}
// 方法引用的变量，可能被其他线程调用，不是安全的
public static void m2(StringBuilder sb){
    sb.append(1);
    sb.append(2);
    sb.append(3);
    System.out.println(sb.toString());
}
// 方法的返回结果可能被其他线程调用，不是线程安全的
public static StringBuilder m3(){
    StringBuilder sb = new StringBuilder();
    sb.append(1);
    sb.append(2);
    sb.append(3);
    return sb;
}

```
## 2.2 栈内存溢出
- 栈帧过多导致栈内存溢出 -- 递归调用
- 栈帧过大导致

```java
/*
  递归调用 案例 导致栈内存溢出
  -Xss256K
 */
public class Demo1_2{
    private static int count;
    public static void main(String[] args) {
        try {
            method1();
        } catch (Exception e) {
            throw new RuntimeException(e);
            System.out.println(count);
        }
    }
    private static void method1(){
        count++;
        method1();
    }
}
```
