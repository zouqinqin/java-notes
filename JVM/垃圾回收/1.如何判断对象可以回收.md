## 1.1 引用计数法
1. 只要对象被引用，计算 +1，如果某个变量不在引用它了，计算 -1，如果最后计算为 0，就可以被回收
2. 有一个弊端，当A 对象 引用B, B又引用A,只要它们相互引用时，就会内存泄漏

## 1.2 可达性分析算法
1. Java 虚拟机中的垃圾回收器采用可达性分析来探测所有存活的对象
2. 扫描堆中的对象，看着否能够沿着GC Root对象作为起点的引用链找到该对象，找不到，便是可以回收

- 哪些对象可以作为GC Root呢？
- 答：可以用Eclipse Memory Analyzer工具 来查看

## 1.3 四种引用
1. 强引用 找不到GC Root时才可以被回收

2. 软引用 在垃圾回收时并且内存不足时 才会被回收（前提是没有强引用）
3. 弱引用 只要发生垃圾回收 就会被回收（前提是没有强引用）
4. 软和弱引用 可以配合引用队列来使用

5. 虚引用必须配合引用队列来使用
6. 终结器引用
