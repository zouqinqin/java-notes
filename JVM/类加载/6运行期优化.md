### 运行期优化

JVM 在运行期间会对热点代码进行即时编译（JIT），将字节码编译为本地机器码，提升执行效率。

---

### JIT 即时编译

- JVM 启动时用**解释器**逐行执行字节码，速度慢但启动快。
- 当某段代码被频繁执行（热点代码），JIT 编译器将其编译为**本地机器码**，后续直接执行，速度大幅提升。
- 热点判断依据：**方法调用次数** 或 **循环回边次数** 超过阈值（默认 10000 次）。

```text
字节码
  ↓ 解释执行（冷启动）
  ↓ 达到热点阈值
  ↓ JIT 编译为机器码
  ↓ 直接执行机器码（热路径）
```

---

### 两种 JIT 编译器（分层编译）

| 编译器 | 特点 | 适用场景 |
|--------|------|----------|
| C1（Client） | 编译快，优化少 | 启动阶段，快速响应 |
| C2（Server） | 编译慢，优化深 | 长期运行的热点代码 |

> JDK 7+ 默认开启**分层编译**：先用 C1 快速编译，再用 C2 深度优化。

---

### 逃逸分析

JIT 的核心优化手段，分析对象的**作用域是否逃出当前方法/线程**。

**不逃逸** → JIT 可以做以下优化：

#### 1. 栈上分配
对象不逃逸时，直接在栈上分配内存，方法结束自动回收，**不进堆，不触发 GC**。

```java
public void foo() {
    // Point 不逃逸出 foo 方法，JIT 可以栈上分配
    Point p = new Point(1, 2);
    System.out.println(p.x + p.y);
}
```

#### 2. 标量替换
把对象的字段拆散，直接用局部变量替代，彻底消除对象分配。

```java
// 优化前
Point p = new Point(1, 2);
int sum = p.x + p.y;

// JIT 标量替换后（等价于）
int x = 1;
int y = 2;
int sum = x + y;
// Point 对象根本没有被创建
```

#### 3. 锁消除
如果对象不逃逸，JIT 发现加的锁不可能被其他线程竞争，直接**去掉锁**。

```java
public String concat(String a, String b) {
    // StringBuffer 不逃逸，synchronized 锁会被 JIT 消除
    StringBuffer sb = new StringBuffer();
    sb.append(a);
    sb.append(b);
    return sb.toString();
}
```

---

### 方法内联

JIT 将频繁调用的**小方法**直接展开到调用处，消除方法调用开销。

```java
// 内联前
int result = add(a, b);

private int add(int x, int y) { return x + y; }

// JIT 内联后（等价于）
int result = a + b;  // 方法调用消失
```

> 内联是 JIT 最重要的优化，很多其他优化（如逃逸分析）都依赖内联后才能生效。

---

### 总结

```text
运行期优化
  ├── JIT 即时编译：热点代码 → 机器码
  ├── 分层编译：C1 快速 + C2 深度
  ├── 逃逸分析
  │     ├── 栈上分配（不进堆，减少 GC）
  │     ├── 标量替换（消除对象创建）
  │     └── 锁消除（去掉无竞争的锁）
  └── 方法内联（消除调用开销，为其他优化铺路）
```
