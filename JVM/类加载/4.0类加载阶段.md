## 类加载有五个阶段
- 一个.class 文件从磁盘到最终被代码使用，要经过五个阶段，这里使用一个比喻：把类加载想象成一个新员工入职的过程

### 4.1 加载 - 收到简历
- 将类的字节码载入方法区中，内部采用C++的instanceKlass 描述java类，它的重要field有：
  * _java_mirror即时java的类镜像，例如对String来说，就是String.lass，作用是把Klass爆漏给java使用
  * _super父类
  * _fields成员变量
  * 。。。。
* 如果这个类还有父类没有加载，会先加载父类
* 加载荷链接可能是交替运行的  
- 注意：
* instanceKlass【元数据】是存储在方法区（1.8的元空间内），但是_java_mirror是存储在堆中
* 可以通过 HSDB 工具查看

### 4.2 验证 - 背景调查
* 验证：验证类是否符合JVM规范，安全性检查


### 4.3 准备 - 分配工位
  - 为static变量分配空间，设置默认值
  - static 分配空间和赋值是两个步骤，分配空间在准备阶段完成，赋值在初始化阶段完成
  - 如果static变量是final的基本类型，以及字符串常量，那么编译阶段值就确认了，赋值在准备阶段完成
  - 如果static变量是final，但属于引用类型，那么赋值发生在初始化阶段

### 4.4解析 - 换工牌
- 将常量池中的符号引用解析为直接引用

### 4.3初始化 - 正式上班
- 这一步才会执行你代码中真正地赋值操作和静态代码块：
```java
public class User {
    static int age = 18;         // 初始化阶段：age 从 0 → 18
    static String name = "Tom";  // 初始化阶段：name 从 null → "Tom"
    
    static {
        System.out.println("类被初始化了");  // 这时候才执行
    }
}
```
- JVM 会把所有 static 变量的赋值语句和 static {} 代码块合并成一个方法叫 <clinit>，在初始化阶段执行它。

- 发生时机
* main方法所在的类，总是会被首先初始化
* 首次访问这个类的静态变量或静态方法时
* 子类初始化，如果父类还没有初始化，会引发
* 子类访问父类的静态变量，只会出发父类的初始化
* Class.forName
* new 会导致初始化

- 不会导致类初始化的情况
* 访问类的static final静态常量（基本类型和字符串）不会触发初始化
* 类对象.class 不会触发初始化
* 创建该类的数组不会触发初始化
* 类加载器的loadClass方法
* Class.forName的参数2为false时
