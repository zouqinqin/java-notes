    public static void main(String[] args) {
        String s1 = "a"; // 懒惰的
        String s2 = "b";
        String s3 = "ab";
        String s4 = s1 +s2 ; // new StringBuilder().append("a").append("b").toString() 相当 new String（“ab”）
        String s5 = "a" +"b"; // javac 在编译器的优化，在编译过程中就明确知道s5的结果是 ab ，然后直接在 StringTable 串池中找
        // 是否存在ab字符串，如果存在就直接取出来，不存在会新增
        System.out.println(s3 == s4);// 结果 false
        System.out.println(s3 == s5); //结果 true
    }

### 5.4 StringTable 特性
1. 常量池中的字符串仅是符号，第一次用到时候才会变成对象
2. 利用串池的机制，来避免重复常见字符串对象
3. 字符串变量拼接的原理是StringBuilder (1.8)
4. 字符串常量拼接的原理是编译期优化
5. 可以使用intern方法，主动将串池中还没有的字符串对象放入串池
  - 1.8将这个字符串对象尝试放入串池，如果有则不会放入，如果没有才会放入串池，并吧串池中的对象返回
  - 1.6将这个字符串对象尝试放入串池，如果有不会放入，如果没有会将堆中的字符串拷贝一份放入串池中，会把串池的对象返回


### 5.6 StringTable 位置
1. JVM 1.6 是常量池的一部分
2. JVM 1.8 是在堆中
