## 排查线程
-[ ] 案例1 ：cpu占用过多

### 定位
1. 用top定位哪个进程对cpu的占用过高
2. ps H -eo pid,tid,%cpu ｜ grep 进程id（用ps命令进一步定位是哪一个线程引起的cpu占用过高）
3. jstack 进程id
   1. 可以根据线程id找到有问题的线程，进一步定位到问题代码的源码行号


-[ ] 案例2 : 程序运行很长时间没有结果
### 定位：
1. 视频是演示了代码中出现死锁情况，例如 a，b线程相互等待对方释放资源

### 案例代码
```java
public class Demo1_3{
    static final A a = new A();
    static final B b = new B();

    public static void main(String[] args) {
        new Thread(()->{
            synchronized (a){
                try {
                    Thread.sleep(2000);
                } catch (InterruptedException e) {
                    throw new RuntimeException(e);
                }
                synchronized (b){
                    System.out.println("获得了 a 和 b");
                }
            }
        }).start();
        Thread.sleep(1000);
        new Thread(()->{
            synchronized (b){
                synchronized (a){
                    System.out.println("获得了 a 和 b");
                }
            }
        }).start();
    }
}


```
